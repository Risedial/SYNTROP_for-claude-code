## Keywords to Avoid

These keywords attract the wrong audience or create misleading expectations:

| Keyword | Why to Avoid |
|---------|-------------|
| "no-code" | This requires CLI comfort — wrong audience |
| "AI writes your code" | Oversells autonomy; sets wrong expectations |
| "fully autonomous" | Not accurate; user approval gates are built in |
| "enterprise AI" | Not positioned or tested for enterprise |
| "replace developers" | Wrong frame; alienates the actual audience |
| "AI software engineer" | Devin owns this territory; creates unfavorable comparison |
| "ChatGPT for coding" | Wrong model, wrong category, dated framing |

## Community Platform Keyword Strategy

### Hacker News

- Use in post title: "state machine," "multi-session," "orchestration," "Claude Code," specific architecture terms
- Avoid in post title: "revolutionary," "next-gen," "AI-powered" (any superlative)

### Reddit (r/ClaudeAI, r/LocalLLaMA, r/MachineLearning)

- Lead with specific problem: "context loss," "multi-session projects," "session boundaries"
- Technical audience — use architecture terms directly

### Dev.to / Hashnode

- Tutorial format: "How I built X using SYNTROP"
- SEO targets: "claude code tutorial," "ai orchestration guide," "multi-session ai development"

### Discord / Slack Communities

- Claude Code community, Anthropic Discord
- Position as "framework for Claude Code users" — emphasize the built-on-top-of relationship
